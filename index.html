Web 퀴즈!

  1. HTTP Method 종류를 아는대로 설명해주세요

    GET
    서버로부터 특정 리소스를 요청, GET 요청은 주로 리소스를 가져오는 데 사용되며, 요청에 데이터를 첨부하지 않습니다.
    HEAD
    서버로부터 특정 리소스에 대한 응답 헤더만을 요청합니다. GET 요청과 유사하지만, 응답 본문을 반환하지 않고 응답 헤더만을 반환합니다.
    POST
    서버에 새로운 데이터를 전송합니다. 주로 폼 데이터나 파일 업로드와 같은 작업에서 사용됩니다. POST 요청은 요청 본문에 데이터를 포함시키며, 이를 통해 클라이언트는 서버에게 정보를 전달합니다.
    PUT
    서버에 새로운 리소스를 생성하거나 기존의 리소스를 업데이트합니다. 요청 본문에 업데이트할 데이터를 포함시킵니다.
    DELETE
    서버에서 특정 리소스를 삭제합니다
    PATCH
    리소스의 일부를 수정합니다. PUT과 유사하지만, PUT은 전체 리소스를 업데이트하는 반면 PATCH는 리소스의 일부를 수정합니다.

  2. 인터넷과 웹의 차이에 대해 설명해주세요.

    Internet
    인터넷은 전세계적으로 컴퓨터와 네트워크 장비들이 연결되어있는 거대한 네트워크 인프라를 의미하며 전세계적으로 데이터를 주곱다는데 사용되는 기술적 기반을 제공합니다.
    Web
    Web은 World Wide Web의 줄임말로, 인터넷 상에서 정보를 공유하고 접근할 수 있는 서비스입니다. HHTP를 통해 데이터를 주고받으며 Html같은 마크업언어를 활용해 문서를 작성해 웹브라우저를 이용해 이를 표시합니다.

    차이점

    인터넷은 전 세계적으로 컴퓨터와 네트워크 장비들이 연결된 거대한 네트워크이고 웹은 인터넷에서 정보를 공유하고 접근할 수 있는 다영한 서비스중 하나이다.

  3. www.naver.com을 치면 무슨 일이 일어날까요?

    통신과정을 크게 3단계로 나눌 수 있습니다.

    1. DNS조회 :

    URL의 도메인 이름을 IP 주소로 변환하기 위해 DNS조회 과정을 거칩니다. 이 과정을 통해 www.naver.com의 실제 서버 위치를 알아냅니다.

    2.TCP/IP 연결 :

    DNS 조회를 통해 얻은 IP 주소로 TCP/IP 연결을 시도합니다.

    HTTP 요청 :

    연결이 성공적으로 이루어지면, 브라우저는 HTTP 프로토콜을 사용하여 서버에 웹 페이지 데이터를 요청합니다.
